# iou - Firebase + RevenueCat Setup & Testing Guide

This document contains the exact steps to wire up the backend services for Google Sign-In + Ad Removal purchases.

## Prerequisites
- A Google account (for Firebase + Google Play)
- An Apple Developer account (recommended for full iOS testing)
- Flutter environment set up
- The code changes from this task are already applied

---

## 1. Firebase Setup

### 1.1 Create / Open Firebase Project
1. Go to https://console.firebase.google.com
2. Create a new project (or reuse an existing one).
3. Recommended name: `iou` or `iou-prod`.

### 1.2 Enable Authentication
1. In Firebase Console → **Authentication** → **Get started**
2. Go to **Sign-in method** tab
3. Enable **Google**
4. Choose a support email
5. Save

### 1.3 Enable Realtime Database
1. Firebase Console → **Realtime Database** → **Create Database**
2. Choose location closest to your users
3. Start in **Locked mode** (we will deploy custom rules next)

### 1.4 Add Android App
1. Firebase Console → Project settings (gear icon) → **Your apps** → Android
2. **Android package name**: `com.gspteck.iou`
3. App nickname: `iou-android`
4. Click **Register app**
5. Download `google-services.json`
6. Place it here:
   ```
   iou/android/app/google-services.json
   ```

### 1.5 Add iOS App
1. In the same "Your apps" section, add iOS app
2. **iOS bundle ID**: `com.gspteck.iou`
3. App nickname: `iou-ios`
4. Download `GoogleService-Info.plist`
5. Add it to Xcode:
   - Open `ios/Runner.xcworkspace`
   - Drag `GoogleService-Info.plist` into the **Runner** folder (not the Tests folder)
   - Make sure it is added to the **Runner** target

### 1.6 Generate `firebase_options.dart`
Run this in the project root:

```bash
# If you don't have FlutterFire CLI yet:
dart pub global activate flutterfire_cli

flutterfire configure
```

Select your Firebase project when prompted.  
This will overwrite `lib/firebase_options.dart` with real values.

### 1.7 Deploy Database Rules (Very Important)
We created a safe minimal rules file at the project root: `database.rules.json`

Run:

```bash
firebase login
firebase init database     # only if you haven't done it before (select Realtime Database)
firebase deploy --only database
```

Or manually paste the contents of `database.rules.json` into the Firebase Console → Realtime Database → Rules tab and publish.

Current rules allow:
- Authenticated users can only read/write their own `/users/$uid/iou` data
- Public read access to the top-level `/iou` node (for future global counters if needed)

### 1.8 Update .env (after you have keys)
See section 3.

## 2. RevenueCat Setup (Ad Removal Purchases)

### 2.1 Create RevenueCat Account & Project
1. Go to https://app.revenuecat.com
2. Sign up / log in
3. Create a new project called `iou`

### 2.2 Add Apps to RevenueCat
- **Google Play**: Add your Android app (`com.gspteck.iou`)
- **App Store**: Add your iOS app (`com.gspteck.iou`)

You will need to connect Google Play and/or App Store later for production.

### 2.3 Create Products

Go to **Products** → **New Product**

Create these two:

| Type          | Identifier               | Name                    | Price (example) | Duration   |
|---------------|--------------------------|-------------------------|-----------------|------------|
| One-time      | `iou_ad_free_lifetime`   | Ad Free (Lifetime)      | $3.99           | Lifetime   |
| Subscription  | `iou_ad_free_monthly`    | Ad Free (Monthly)       | $0.99           | Monthly    |

### 2.4 Create Entitlement
1. **Entitlements** → **New Entitlement**
2. **Identifier**: `Ad Free`   ← **Must be exactly this string**
3. Attach both products above to this entitlement.

### 2.5 Create an Offering + Paywall
1. **Offerings** → **New Offering**
2. Name it something like `Default` or `AdFree Offering`
3. Add the two products to the offering
4. Create a **Paywall** (you can use the default template or a simple one)
5. Save and publish

RevenueCat will now serve this paywall when the app calls:
```dart
RevenueCatUI.presentPaywallIfNeeded('Ad Free');
```

### 2.6 Get API Keys
In RevenueCat project settings:
- Copy the **Google API Key**
- Copy the **Apple API Key**

Put them in `.env` (see below).

## 4. Testing the Full Flow

### 4.1 Android Testing (Recommended first)

#### Prerequisites for real purchases
- You need a **Google Play Console** account
- Create an **Internal testing** track (or Closed test)
- Upload at least one signed APK/AAB (even if just for testing)
- Add your test Gmail account(s) as testers
- Accept the tester invitation email on the device

#### Quick local test (limited)
You can test the UI flow without a published app using RevenueCat's "Override" feature or by using a real test purchase after the app is in a track.

#### Step-by-step test on device

1. **Install the app** on a physical Android device (emulators have limited Google sign-in).

2. **First run (no login)**:
   - Tap the "+" (Add Button)
   - An interstitial ad **should appear** (Appodeal)

3. **Sign in**:
   - Open Settings (gear icon in top bar)
   - Tap "Sign in with Google"
   - Complete Google sign-in
   - You should see your avatar + "Logged in"

4. **Purchase Ad Removal**:
   - In Settings, tap **"Disable Ads • 3.99 or 0.99/mo"**
   - RevenueCat paywall should open
   - Choose either the one-time or monthly option
   - Complete the **sandbox / test purchase**

5. **Verify ads are disabled**:
   - Close Settings
   - Add a new person (tap +)
   - **No interstitial** should show
   - Check logs or add temporary prints if needed

6. **Verify Firebase**:
   - Go to Firebase Console → Realtime Database
   - Look for path: `users/{your-uid}/iou/adFreeStatus` → should be `true`

7. **Sign out test**:
   - In Settings tap "Sign Out"
   - Add another person → **ad should appear again**

8. **Restart app test**:
   - Force close and reopen the app while logged in
   - `isAdFreeActive()` runs on a timer — ads should stay disabled

### 4.2 iOS Testing

Similar flow, but:
- Use **TestFlight** or a development build with a real device
- Use Apple Sandbox accounts (create in App Store Connect → Users and Access → Sandbox)
- Sign in to the device with the Sandbox Apple ID before testing purchases
- RevenueCat will automatically use sandbox when the app is not production-signed

### 4.3 Useful RevenueCat Testing Tools
- In RevenueCat dashboard you can see customer info in real time
- Use "Customer" search by the Firebase UID
- You can manually grant/revoke the "Ad Free" entitlement for testing

### 4.4 Common Issues & Fixes

| Problem                          | Likely Cause                              | Fix |
|----------------------------------|-------------------------------------------|-----|
| Paywall does not open            | Wrong entitlement name or no offering     | Must be exactly `"Ad Free"` |
| Purchase fails immediately       | App not in a Google Play test track       | Create internal test track + add tester |
| Still seeing ads after purchase  | Status not yet polled or `adFree` not set | Wait 2-3 seconds (polling interval) or restart |
| Google Sign-in fails             | `google-services.json` missing or wrong package name | Double-check package is `com.gspteck.iou` |
| Rules permission error           | Rules not deployed or too strict          | Deploy the `database.rules.json` we created |
| No products in paywall           | Offering not published or products not attached | Check RevenueCat dashboard |
